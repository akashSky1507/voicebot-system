Latency figures are typical published/observed p50 ranges as of mid-July 2026, for modeling only — always measure p50/p95 against your own real call audio and network path before committing to a stack. Non-streaming ("batch"/"pre-recorded") STT and TTS options are flagged: they work in this calculator to show the cost of using them in a live call, but they are not suitable for a real-time voicebot. GPT-5.4's default reasoning behavior is flagged the same way: it's a strong model but not built for a live conversational turn unless you use a fast variant (nano) or set reasoning effort to none/low. STT, LLM, and TTS are independent microservices the orchestrator reaches over a persistent WebSocket connection each — the "WebSocket hop latency" input models that network round-trip cost, applied once per service (3× total), on top of each vendor's own quoted latency.
